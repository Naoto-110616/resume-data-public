# 転職選考 - 共通知識 vs 企業別知識フレームワーク

**作成日:** 2026-05-09  
**目的:** 複数企業の面談準備において、「全企業で共通して覚えるべきこと」と「企業ごとにカスタマイズすべきこと」を整理する

---

## 📊 全体マップ

```
【転職選考の準備】
│
├─ 【共通知識】✅ 1回の準備で複数企業に応用可能
│  ├── 面談の流れ・心構え
│  ├── 自分自身の基礎情報（経歴、スキル、実績）
│  ├── 面接テクニック（STAR法則など）
│  ├── 一般的な予想質問への考え方
│  └── 逆質問のフレームワーク
│
├─ 【企業別知識】🎯 企業ごとに 100% カスタマイズ必要
│  ├── 企業基本情報（設立、規模、事業内容）
│  ├── 企業文化・ビジョン・バリュー
│  ├── 採用職種の詳細要件
│  ├── 競合他社との違い
│  ├── あなたとの詳細マッチング分析
│  └── 企業別カスタマイズ自己紹介
│
└─ 【面談直前チェック】🔔 企業別の最終確認
   ├── ポートフォリオ・GitHub確認
   ├── 技術例の準備
   └── 逆質問の最終厳選
```

## 1️⃣ 【共通知識】全企業で共通して覚えるべきこと

### 1-1. 面談の流れ・心構え

| 項目 | 内容 | 学習方法 |
|------|------|--------|
| **カジュアル面談の目的** | 相互理解（採用側：あなたを知る、あなた側：企業を知る） | `08_カジュアル面談を効果的に行うポイント.md` |
| **構成時間** | 30-60分（導入5分+会話25-45分+逆質問10分） | 経験から学習 |
| **面談官の心理** | 「一緒に働けるか」「本当に興味あるのか」を見ている | 「面接で通る方の共通項.md」参照 |
| **成功の指標** | 相手の回答で「初めて知ること」が1個以上ある逆質問ができたか | フィードバック記録 |

**固定テンプレート例（Scene Liveで実証済み）:**
```
【導入】
- 自己紹介（2分）
- 企業への興味表明（1分）

【会話パート】
- 相手の質問に STAR 法則で回答（20-30分）
- あなたが主体的に質問を混ぜる（5-10分）

【逆質問】
- 準備した TOP 3 から選定（5-10分）

【クロージング】
- お礼の言葉（1分）
```

### 1-2. 自分自身の基礎情報（一度準備したら複数企業で使用）

#### A. 経歴・職務内容の整理

**一度準備すれば、全企業で同じ内容をベースに使用できる：**

```
【職務経歴（2020年～現在）】

📌 前職（2020年4月～2024年12月）
- 職務: フロントエンドエンジニア（React/TypeScript）
- 規模: 4名のエンジニアリングチーム
- 実績: 
  * ページロード速度 40% 改善（3秒→1.8秒）
  * Figmaでの設計→実装の一貫対応で開発速度 2倍
  * 新人オンボーディングマニュアル作成

📌 現職（2025年1月～現在）
- 職務: シニアフロントエンドエンジニア
- 規模: 8名チーム（2名マネジメント経験）
- 実績:
  * デザイン→実装の品質基準確立
  * CI/CD 自動化で デプロイ時間 70% 削減
  * ...
```

**使用場面:**
- ✅ 全企業の面談で同じ内容で OK
- ⚠️ 企業の評価軸に合わせて「どの実績を強調するか」はカスタマイズ
  - スタートアップ → スピード感、自主性を強調
  - 大企業 → チームマネジメント、品質基準を強調

#### C. マネジメント経験の言語化

**全企業で共通に用意する基本形：**

```
【マネジメント経験】

📌 チーム体制

   → 判断基準をドキュメント化
   → GitHub Issues でのコメント駆動開発

② 個別 1on1 ミーティング（隔週）
   → キャリア相談
   → 技術的な課題解決

③ 新人オンボーディング
   → マニュアル作成（20ページ）
   → ペアプログラミング導入

📌 成果
- チームの離職率 0%
- スキル習得時間 平均 3ヶ月（業界平均 6ヶ月）
- リーダーシップ事例...
```

**使用場面:**
- ✅ 基本形は全企業で同じ
- ⚠️ 企業の組織規模に合わせてアレンジ
  - スタートアップ（5-20名） → 「3名のチーム」という小さいパターン
  - 大企業（100名+） → 「複数プロジェクトへの関与」を強調

### 1-3. 面接テクニック（全企業共通）

#### STAR 法則

**回答の構造（全企業で同じ）:**

```
【S】Situation（状況）
 →「当時、〇〇というプロジェクトをやっていて...」

【T】Task（課題）
 →「△△という課題がありました」

【A】Action（行動）
 →「〇〇という行動を取りました」

【R】Result（結果）
 →「□□という成果を得ました」
```

**具体例（Scene Live での実装）:**
```
Q: 「チーム開発で工夫していることは？」

S: 前職では 4名のフロントエンドチームを担当していました
T: 非同期チーム（タイムゾーン差 +3時間）での
   判断遅延が課題でした
A: 判断基準をドキュメント化し、GitHub Issues で
   コメント駆動開発を導入しました
R: 判断時間が 70% 短縮され、チーム満足度が向上しました
```

**使用場面:**
- ✅ 全企業で同じ構造を使用
- ⚠️ 同じ経歴でも「企業が重視する軸」に合わせて
  どの実績を選ぶかはカスタマイズ

---

#### コミュニケーションの基本テクニック

| テクニック | 説明 | 例 |
|----------|------|-----|
| **具体性** | 「改善した」ではなく数字を入れる | 「開発時間を1日→半日に短縮」 |
| **私が主語** | 「うちの会社では」ではなく「私は〜しました」 | × 「うちではアジャイル」→ ○「私がアジャイル導入を...」 |
| **相手を観察** | 相手の回答から「実は課題かな」と感じたことを質問 | 相手が「チームのコミュニケーション」に触れたら、その課題について逆質問 |
| **完璧を目指さない** | 「その部分はわかりません」は誠実 | 「具体的には...」と無理に答えない |

---

### 1-4. 予想質問への考え方フレームワーク

**全企業で共通の問い（ただし回答の詳細はカスタマイズ）:**

#### Q1: 「なぜこの企業に興味を持った？」
- **共通フレーム:** 企業の〇〇という価値観・プロダクト・技術スタックが、自分のキャリアの 〇〇と合致している
- **カスタマイズ:** 企業ごとに「〇〇」の部分を変える

#### Q2: 「これまでで最も成長した経験は？」
- **共通フレーム:** 課題 → 行動 → 結果（STAR法則）で語る
- **カスタマイズ:** どの経験を選ぶか（企業の価値観に合った経験を選定）

#### Q3: 「弱みや課題は？」
- **共通フレーム:** 「改善中の課題」として答える（= 弱点ではなく「成長機会」として語る）
- **カスタマイズ:** 同じ弱みでも、企業の技術スタックに合わせた改善方向を示す
  - Python スキル不足 → Python 企業に対しては「FastAPI の実装経験を積んでいる」と向き合っている姿勢を示す

#### Q4: 「今後のキャリアは？」
- **共通フレーム:** 「技術的成長」と「人への影響」のバランスを示す
- **カスタマイズ:** 企業の成長段階に合わせてトーンを変える
  - スタートアップ → 「スケーリング段階での技術基盤構築に貢献したい」
  - 大企業 → 「深い専門性を持ったアーキテクトになりたい」

---

### 1-5. 逆質問のフレームワーク

**全企業で共通のカテゴリ構成：**

```
【カテゴリ A】チーム・文化について
- 「エンジニアリング組織の課題として、今最も焦点が当たっているのは...」
- 「チーム間のコラボレーション」に関する質問

【カテゴリ B】技術・開発について
- 「今後 12 ヶ月で導入予定の技術スタック...」
- 「レガシー コード の改善計画...」

【カテゴリ C】職務・成長について
- 「このポジションで最初の 3 ヶ月で期待される成果...」
- 「1年後のキャリアイメージ...」
```

**カスタマイズ:** 
- 共通カテゴリは同じだが、優先度付けは企業によって変更
- 企業サイズ・段階 → 聞きたい質問の優先順位が異なる

## 2️⃣ 【企業別知識】企業ごとに 100% カスタマイズが必要

### 2-1. 企業基本情報の収集

**必須項目（全企業で共通の収集項目）:**

| 項目 | 詳細 | 入手元 |
|------|------|------|
| **基本情報** | 設立年、社員数、売上、本社所在地 | 公式HP / Wantedly |
| **事業内容** | メインサービス、機能、ターゲット顧客 | 公式HP / ブログ |
| **組織構成** | エンジニア人数、マネジメント体制、採用中職種 | Wantedly / 求人ページ |
| **技術スタック** | 使用言語、フレームワーク、インフラ | GitHub / ブログ / 面談官のプロフィール |
| **成長・資金調達** | 最新の資金調達、ターゲット成長率 | TechCrunch / プレスリリース / Crunchbase |
| **採用ページの価値観** | 企業が何を大事にしているか | Wantedly / 採用ページ / ブログ |

**Scene Live の例:**
```
✅ 設立: 2011年 / 社員: 73名 / 本社: 大阪
✅ メイン事業: lisnavi（採用管理SaaS）
✅ 最新資金調達: Series A
✅ 価値観: 「個人の成長」「顧客起点」「透明性」
✅ エンジニア: 約 15名 / React 中心
```

---

### 2-2. あなたと企業のマッチング分析

**各企業ごとに実施（スキル・カルチャー・キャリア軸）:**

#### テンプレート:

```
【スキルマッチング】
企業が求める: React / TypeScript / Next.js
あなたの実績: ⭐⭐⭐⭐⭐ (5年の実装経験)
→ マッチング度: 非常に高い
→ 面談での話し方: 「これが最大の強みです」と前面に出す

企業が求める: Python / FastAPI
あなたの実績: ⭐⭐⭐ (1年の実装経験)
→ マッチング度: 中程度
→ 面談での話し方: 「フロントエンドを主軸としながら、バックエンド
  との協業経験を積んでいます。FastAPI への学習意欲も高い」

【カルチャーマッチング】
企業の価値観: 「スピード重視」「個人の自主性」
あなたの性質: スタートアップで意思決定 → 実装を最速で行う
→ マッチング度: 高い
→ 面談での話し方: スタートアップでの「判断→実装→検証」のサイクルを強調

企業の価値観: 「深い専門性」「複数年のプロジェクト」
あなたの性質: スタートアップで「広く浅く」の経験
→ マッチング度: 中程度
→ 面談での話し方: 「長期的な基盤構築への興味」と「既存経験の応用」を組み合わせる

【キャリアマッチング】
企業が期待する成長: 「アーキテクトへの昇進」
あなたのキャリア希望: 「技術的深さ」「チームへの影響」
→ マッチング度: 高い
→ 面談での話し方: 「貴社でのアーキテクト育成制度に惹かれた」と表現
```

**→ このマッチング分析が「企業別カスタマイズ自己紹介」の基盤になる**

### 2-3. 企業別カスタマイズ自己紹介

**基本構造（全企業共通）:**
```
【30秒版】
名前 + 職務 + 得意なこと + その企業への興味（1文）

【60秒版】
(30秒版) + 具体的な実績 1 個

【2分版】
(60秒版) + 経歴 + キャリア軸の説明
```

**Scene Live への例：**
```
【2分版】
「大川尚斗です。フロントエンドエンジニアで、React / TypeScript / Next.js
での実装と Figma でのデザイン設計を得意としています。

前職では 4 名のフロントエンドチームを担当し、ページロード速度を
40% 改善、チーム間のコミュニケーションスピードを 70% 短縮した経験
があります。

特に『個人の自主性を尊重しながら高速に成長する』という Scene Live
の価値観に強く共感し、スタートアップでの経験を活かしながら、
貴社のプロダクト成長に貢献したいと考えています。」
```

**他企業へのカスタマイズ例：**
- **大企業 SaaS 企業**の場合 → 「深い専門性」「チームマネジメント」を強調
- **バックエンド 企業**の場合 → Python / FastAPI 経験を前面に

### 2-4. 企業別の予想質問と回答

**共通質問の基本形は同じだが、企業別にカスタマイズ：**

#### Q1: 「なぜこの企業に興味を持った？」

**共通フレーム:**
→ 企業の〇〇（特徴・価値観・プロダクト）と、自分の〇〇（経験・スキル・キャリア軸）が一致している

**Scene Live の場合:**
```
「採用管理の複雑性を解決する lisnavi というプロダクトに感動しました。
採用は会社の成長を左右する機能ですが、現状のツールは使い勝手が
悪い。貴社が UX を重視しながら解決している姿勢に惹かれました。

また、『個人の自主性』という価値観が、スタートアップでの
自分の経験と完全に一致しています。」
```

**別企業例（大企業の場合）:**
```
「貴社の長期的なビジョンと、複数プロジェクトへの長期関与を通じて
深い専門性を育成する環境に惹かれました。スタートアップでの
『広く浅く』の経験から、今後は『深く狭く』の専門性を磨きたいと
考えており、貴社がその機会を提供してくれると確信しています。」
```

```
【職務・成長】
```

## 3️⃣ 【準備プロセス】どう使い分けるか

### ステップ 1: 共通知識の一度の準備（最初だけ）

**実行内容:**
```
□ 自分の経歴を整理（職務内容、スキル、実績）
□ マネジメント経験を言語化
□ STAR 法則の練習
□ 予想質問の考え方フレーム作成
□ 逆質問プール（30個以上）を作成
```

**所要時間:** 5-6 時間（最初だけ）

**保存場所:** `/documents/転職tips/` または `/documents/共通知識/`
（全企業で参照できるようにドキュメント化）

---

### ステップ 2: 企業別知識の準備（企業ごと）

**実行内容:**
```
□ 企業基本情報の収集（1 時間）
□ あなたとのマッチング分析（30 分）
□ 企業別カスタマイズ自己紹介を作成（30 分）
□ 企業別の予想質問 TOP 5 を作成（20 分）
□ 逆質問 TOP 3 を選定（15 分）
```

**所要時間:** 2.5-3 時間 / 企業

**保存場所:** `/企業別面談準備/[企業名]/`

---

### ステップ 3: 面談直前（企業別、面談の 24 時間前）

**実行内容:**
```
□ ポートフォリオ・GitHub の最終確認
□ 技術デモの準備
□ 逆質問 TOP 3 の最終レビュー
□ 自己紹介の音読練習
```

**所要時間:** 1-2 時間

## 4️⃣ 【ドキュメント構成の推奨】
