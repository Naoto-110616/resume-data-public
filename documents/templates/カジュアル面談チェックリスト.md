## ① 第一志望化に向けた準備

**目的**：カジュアル面談を選考につなげる。面談終了時に「第一志望」と思える状態を目指す

### チェックリスト

- [ ] **企業情報の理解**
  - [ ] 企業のHP・採用ページを読んだ
  - [ ] 求人票を読んだ
  - [ ] 実際の事業内容・サービスを理解した
  
- [ ] **主観的な興味の言語化**
  - [ ] なぜこの企業に応募したのか？（3つ以上） を書き出した
  - [ ] その興味が本当か、表面的でないか自問自答した
  
- [ ] **不安・モヤモヤポイントの整理**
  - [ ] 入社を躊躇させる要素を2-3個書き出した
  - [ ] 各々について「カジュアル面談で何を質問したら解決するか」を明記した
  
  例）
  - モヤモヤ：新規事業が立ち上がったばかりだから不安定かも → 質問：「新規事業のロードマップと経営層の本気度は？」
  - モヤモヤ：技術スタックが古いと聞いた → 質問：「今後の技術刷新の計画は？」

### 合格ライン

✅ **クリア判定**：主観的な興味 3 個 + 解決したいモヤモヤ 2-3 個が、**具体的に**書かれている状態

### 確認方法

- 書き出した内容を読み返して、「採用面接で話しても恥ずかしくない具体度」があるか判定
- 曖昧な表現（「成長できそう」「やりがいがありそう」など）がないか確認

## ② メモ・コミュニケーション準備

**目的**：面談を振り返る際、また選考対策時に活用できるメモを取る

### チェックリスト

- [ ] **メモツール・方法の決定**
  - [ ] 紙とペン / ノート / iPad / PC のどれを使うか決めた
  - [ ] フォーマットを軽く決めた（例：タイムライン形式 / トピック別 / QA形式）

- [ ] **心理学的テクニックの準備**
  - [ ] 担当者のお名前をメモするスペースを用意した
  - [ ] 面談中に担当者名を呼びながら会話する「イメージトレーニング」をした
  - [ ] 担当者の表情・反応を観察することを意識した

- [ ] **メモ項目の事前設計**
  - [ ] 質疑応答の内容を記録する枠を用意した
  - [ ] 担当者のお名前・役職・雰囲気を書く欄を用意した
  - [ ] 「ここが意外だった」「ここが響いた」という感想欄を用意した

### 合格ライン

✅ **クリア判定**：メモツールが決まっており、**具体的な記入枠**を設計済み

### 確認方法

- 実際に面談を想定して、ツール・フォーマットで 1-2 項目書いてみる
- 違和感がないか、必要な情報が抜け落ちていないか確認

## ③ 企業の課題感ヒアリング準備（重要）

**目的**：採用背景にある「企業の課題・課題」を引き出す。これが志望動機・自己PRの強力なネタになる

### チェックリスト

- [ ] **「なぜこのタイミングで採用?」を考える**
  - [ ] 企業が抱えている課題・伸ばしたい事業を3個推測した
  - [ ] HP・求人票には明記されていない「奥にある課題」を想像した
  
  例）
  - 表面：「Next.js での FE 開発者募集」
  - 奥：「既存の jQuery コードベースからの移行を進めたい」「フロントエンドの品質向上が課題」

- [ ] **企業の課題に対する「ヒアリング質問」を用意**
  - [ ] 5-7 個の「課題感」を引き出す質問を書き出した
  - [ ] 各質問が「コンサルティング的」か（相手の本音を引き出すか）確認した
  
  良い例）「現在チームが抱えている技術的な課題は何ですか？」
  悪い例）「どんな技術を使ってますか？」

- [ ] **ヒアリング後のアクション設計**
  - [ ] 「課題に対して、自分の経歴・実績がどう貢献できるか」をイメージしておいた
  - [ ] 一次選考までに「課題 → 自分の貢献シナリオ」を言語化する計画を立てた

### 合格ライン

✅ **クリア判定**：推測した企業課題 3 個 + ヒアリング質問 5-7 個が準備済み

### 確認方法

- ヒアリング質問を読み返して「相手が『実はね…』と本音を言いたくなる質問か」判定
- 企業のビジネス情報から推測を確認「この推測は妥当か」を確認

## ④ 最後の印象を残す準備

**目的**：カジュアル面談のラストで良い印象を残す。記憶に残る終わり方を用意する

### チェックリスト

- [ ] **一般的な「つまらない終わり方」を避ける意識**
  - [ ] 「他にご質問ありますか」→「大丈夫です」という無難なパターンを避ける
  - [ ] なぜ避けるべきか理解した（最後は最も記憶に残るため）

- [ ] **良い印象を残す「締めの言葉」を用意**
  - [ ] 面談を通じて得た学びを1つ言語化した
  - [ ] 「意外だったこと」「響いたことば」を1-2個用意した
  - [ ] 時間を割いてくれたことへの**心からの感謝**を言葉にした
  
  例）「本日はお時間をいただきありがとうございました。特に◇◇の話が印象的で、△△という学びが得られました。引き続き貴社のことを勉強させていただきたいと思います」

- [ ] **温度感判定**
  - [ ] 企業の「温度感」（カジュアルさ度）を認識した
  - [ ] 温度感が高い企業なら、スタンスを「形式的になりすぎない」に調整した
  - [ ] 温度感が低い企業なら「きちんと感」を大事にした

### 合格ライン

✅ **クリア判定**：**実際に言える締めの言葉**が用意済み（暗唱できるレベル）

### 確認方法

- 鏡の前で、または知人相手に、用意した「締めの言葉」を実際に言ってみる
- 自然に聞こえるか、作り込まれすぎていないか確認

## ⑤ 年収・グレード感のヒアリング準備

**目的**：条件面での齟齬を事前に防ぐ。お見送りになる理由を減らす

### チェックリスト

- [ ] **現状の年収・条件を整理**
  - [ ] 現職の年収を把握している
  - [ ] 希望年収の根拠を持っている
  - [ ] 自分の「市場価値」を冷静に認識している

- [ ] **企業側のレベル感をヒアリングする質問を用意**
  - [ ] 「この職種のレベル感は？」を聞く質問を用意した
  - [ ] 「現在年収 ◇◇ですが、貴社ではどのグレードに該当しそうですか？」と確認する流れを想定した
  
  例）
  - 「同じポジションで同等の経験を持つ方は、社内ではどのグレード・職級に位置づけられていますか？」
  - 「そのグレードの期待値・役割・年収レンジはどのくらいですか？」

- [ ] **レベル別の期待値を理解する**
  - [ ] 現状と同じグレードに求められる役割・実績を質問リストに入れた
  - [ ] 「次のグレード」への期待値も聞く流れを想定した
  - [ ] 「年収が上がるとしたら、どんな条件が揃えば？」も質問に入れた

- [ ] **事後対応の準備**
  - [ ] 面談後、ヒアリング内容を「年収期待値シート」に記録する計画を立てた
  - [ ] 複数企業の情報を比較する仕組みを用意した

### 合格ライン

✅ **クリア判定**：グレード感・年収レンジを聞く質問 3-4 個が準備済み、比較記録方法も決定済み

### 確認方法

- 「現年収 ◇◇だが、貴社ではどのグレード？」という質問を、自然に会話に組み込めるか確認
- 複数企業の情報を記録・比較する仕組みが機能するか確認
