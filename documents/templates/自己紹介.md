# 自己紹介スクリプト（カジュアル面談用）

カジュアル面談・選考面接の冒頭で使う自己紹介の原稿。

冒頭で「簡単に自己紹介をお願いします」と聞かれた際に、迷わず話せるようテンプレート化したもの。状況に応じて 1分30秒版 / 45秒版 を使い分ける。

## 1分30秒版（標準）

```text
改めて、大川尚斗と申します。本日はお時間をいただきありがとうございます。

現在は株式会社アトムズテクノロジーで、フロントエンドエンジニアとして
Next.js / TypeScript を中心とした開発に従事しています。

エンジニアとしてのキャリアは2021年からで、現在で約5年になります。
最初はWordPressでのコーポレートサイト制作からスタートし、
現職ではShopify PlusでのEC開発、Next.jsでの新規プロダクト開発、
Laravelでの API実装まで幅広く担当してきました。

実績として印象的なのは、大規模ECリプレイス案件で
コンポーネント設計と開発フロー改善を担当し、
1ページあたりの実装時間を1日から半日に、約50%短縮できたことです。
直近では4名のメンバーをマネジメントするリーダーポジションも経験し、
顧客折衝・技術選定・コードレビューまで一貫して担当しています。

技術面の強みは、Figmaのデザインを読み取りながら Next.js での実装、
本番デプロイまでを一貫して担当できることです。
既存のデザインシステムの中でFigmaを更新しながら実装まで担えるため、
デザインとエンジニアリングを横断できる点が自分の強みだと考えています。

今後はUI/UXを起点に価値を生み出せるエンジニアを目指しており、
本日はそういった環境について、お話を伺えればと思っております。
よろしくお願いいたします。
```

### 深掘りされた際の定量実績メモ

「もう少し具体的に」と聞かれたときに使えるエピソード（どれも深掘り歓迎のレベルで準備する）：

- **イオンリテール ECリプレイス**：Figma × GitHub Issue 連携で実装着手コストを削減し、1ページあたりの実装時間を約50%短縮（1日→半日）
- **Shopify Plus リーダー経験**：4名マネジメント、顧客折衝〜オフショア管理まで担当。自動デプロイフロー整備でデグレ件数をほぼゼロに抑制
- **美容室サイトリプレイス（自己研鑽）**：年間ランニング費用をドメイン代のみに削減、運用保守契約も獲得

### 構成解説

| パート | 内容 | 目安秒数 |
| --- | --- | --- |
| ① 挨拶 | 名前と感謝 | 5秒 |
| ② 現職 | 何をしているか | 15秒 |
| ③ キャリア概要 | これまでの経験の幅 | 25秒 |
| ④ 強み | 一番のアピール | 25秒 |
| ⑤ 締め | 今後と面談への期待 | 15秒 |

## 45秒版（短縮）

「短くお願いします」と言われた時用の予備バージョン。

```text
改めて、大川尚斗と申します。本日はよろしくお願いいたします。

2021年からフロントエンドエンジニアとしてキャリアをスタートし、
現在で約5年になります。
直近ではNext.js / TypeScriptを中心に、
ECサイトや新規プロダクトの開発を担当しています。

強みはFigmaと連携しながら実装まで一貫して担当できることで、
デザインとエンジニアリングを横断できる点を活かしてきました。

本日はお話を伺えるのを楽しみにしておりました。
よろしくお願いいたします。
```

## 話し方のコツ

### やること

- **最初の画面切り替え時に笑顔で挨拶する**（第一印象はここで決まる）
- 棒読みしない（自然に話す）
- 間を取る（早口にならない）
- 目線をカメラに（画面の人物ではなくレンズ）
- 笑顔を意識（リモート面談は表情が伝わりにくい）
- **声に出して練習する**（準備した内容は必ず声に出して口に馴染ませる。頭の中で読むだけでは本番で自然に出てこない）

### やらないこと

- 暗記して呪文のように話す
- 1分以上かけて話す（相手が飽きる）
- 専門用語を多用する（採用担当者がエンジニアじゃない場合もある）

## カスタマイズポイント
