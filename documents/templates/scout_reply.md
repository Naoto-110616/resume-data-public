# スカウト返信テンプレート

Findy等のスカウトメッセージへの返信用テンプレート集。3つのパターンを使い分ける。

## 想定シーン

| パターン | 状況 | 使用頻度 |
| --- | --- | --- |
| ① カジュアル面談を希望 | 興味あり、まず話を聞きたい | 一番多い |
| ② 詳しい情報を聞きたい | 気になるけど判断材料が足りない | 時々 |
| ③ 丁寧にお断り | 興味なし or 条件が合わない | 時々 |

## ① カジュアル面談を希望（メイン）

```text
〇〇様

お忙しいところメッセージをいただき、ありがとうございます。
大川と申します。

ご紹介いただいた〇〇のポジションについて、
ぜひ一度カジュアル面談でお話を伺えればと考えております。

現在、転職活動を始めたところで、まずは情報収集として
様々な企業様のお話をお伺いしている段階ですが、
御社の〇〇という点に興味を持ちました。

下記の日程で都合の良い時間がございましたら、ご教示いただけますと幸いです。

【希望日程】
・〇月〇日（月）14:00〜18:00
・〇月〇日（火）10:00〜12:00、15:00〜18:00
・〇月〇日（水）終日可

オンラインでの実施を希望しております。
何卒よろしくお願いいたします。

大川 尚斗
```

## ③ 丁寧にお断り

```text
〇〇様

メッセージをいただき、ありがとうございます。
大川と申します。

ご提案いただいたポジションについて、内容を拝見いたしました。
大変魅力的なお話ではございますが、
現在私が考えている方向性とは少し異なるため、
今回は見送らせていただければと存じます。

せっかくお声がけいただきましたのに、ご期待に添えず申し訳ございません。
今後ともよろしくお願いいたします。

大川 尚斗
```

## 必ず変える箇所

| 変更箇所 | 内容 |
| --- | --- |
| `〇〇様` | 担当者名 |
| `〇〇のポジション` | 求人タイトル |
| `〇〇という点` | 興味を持った具体的な点 |
| `【希望日程】` | 自分の都合 |

## 「興味を持った点」の書き方例

スカウトメッセージや求人内容から、1つだけ具体的に選んで書く。

| 求人タイプ | 書き方例 |
| --- | --- |
| デザインエンジニア系 | デザインとエンジニアリングを横断する役割を募集されている点 |
| Next.js × AI系 | Next.jsを軸に生成AIを活用した開発を進めている点 |
| フルリモート | 地方フルリモートでの働き方が前提となっている点 |
| スタートアップ | 新規プロダクトを立ち上げフェーズから関われる点 |
| 大手・SaaS | 安定した事業基盤の上で新しい技術にも挑戦されている点 |
| EC | 大規模なECサイトを支えるフロントエンド技術への取り組み |

## 注意点

- **コピペ感を出さない** → 「興味を持った点」だけは必ず個別に書く
- **日程は3〜4枠提示** → 1枠だけだと相手が困る
- **時間帯は現職を考慮** → 平日夜（19時〜）も入れると親切
- **送信前に必ず読み返す** → 「〇〇」が残っていないかチェック

## 練習用3社への適用例

### Scene Live 向け

> 御社の新規AIプロダクト開発において、フルスタックでの開発を進めている点

### Red Frasco 向け

> 不動産という大きな市場で、Next.jsを用いてサービス成長を支えている点

### メディア工房 向け

> リモートメインで上場2社目という環境で、フロントエンドの中心メンバーを募集されている点

## メッセージ送信時のチェックリスト

- [ ] 企業/人物のリサーチは十分か（具体性があるか）
- [ ] 「〇〇」や「△△」を全て埋めたか
- [ ] 敬語・文体は統一されているか
- [ ] 長すぎないか（100〜200語程度が目安）
- [ ] 相手にとってメリットが見えるか
- [ ] 送信日時は適切か（平日営業時間がベター）
- [ ] 人柄が少し出ているか（完全テンプレ感がないか）

## 送信前のコツ

- **具体性が命** → 企業/プロダクトの「何が好きか」を必ず書く
- **短すぎず、長すぎず** → 返信しやすい長さを意識
- **セールスっぽくない** → 「一緒に話しましょう」スタンスで
- **フォローアップ** → 1週間返信なければ、別のアプローチ検討
- **複数送信** → 一度に3〜5社程度が目安（高頻度は避ける）
